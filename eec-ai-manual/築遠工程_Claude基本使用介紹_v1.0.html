PART 01

先認識工具

在點開 Claude 之前,先弄清楚:你用的是哪一套、跟「個人付費版」有什麼不同、登入要用哪個信箱。看似瑣碎,但弄錯就是後面所有規範的破口。

§ 01

Claude 是什麼,我們用的是哪個版本

Claude 是 Anthropic 公司開發的生成式 AI 助理,可透過網頁版(claude.ai)與手機 App(iOS / Android)使用。公司採用 Team 方案;與個人付費的 Pro 方案相比,差別在用量配額、可共用 Project、以及帳號由公司統一管理。

同仁日常會接觸到的 Claude 模型有三個系列,可在輸入框旁邊切換:

模型適用場景用量耗用
Opus較強、較慢,適合複雜推理、長文件、需要嚴密邏輯高
Sonnet平衡型,適合日常使用、報告撰寫、規範整理中
Haiku輕量、快速,適合短問答、格式整理低

公司並未強制使用哪一個;但每個模型的用量計算方式不同,日常工作優先選 Sonnet 通常就夠用(見 § 09)。

Note · 用語對照
本文使用「對話」「Project」「Artifact」「知識庫」這幾個詞,皆為 Claude 介面上的正式用語,並非公司自創。後續細則或公告若出現相同用詞,意義一致。
§ 02

登入與初次設定

2-1 · 第一次登入
1
收到管理單位通知
確認用來開通的是公司網域信箱。打開瀏覽器(建議 Chrome 或 Edge),前往 claude.ai。
2
選擇登入方式
選「以 Google 登入」或「以 Email 登入」,輸入公司網域信箱。不要使用個人 Gmail 或私人信箱。
3
加入公司 Team 組織
第一次登入時,系統會將你加入公司 Team。畫面左上角會顯示組織名稱,確認是「築遠工程」而非個人帳號。
4
基本設定
完成個人資料設定。語言可設為「繁體中文」;其他預設值先不用動,熟悉後再調整。
2-2 · 個人帳號 vs 公司帳號的辨認

若你原本就有私人的 Claude 帳號,登入後請確認左上角的組織標示。

Company · 公司帳號
左上角顯示組織名稱

顯示「築遠工程」或類似的組織標識。屬於 Team 方案,可建立共用 Project、可被管理單位管理。

涉及公司業務的 AI 使用,一律走這裡。

Personal · 個人帳號
只顯示你的名字或頭像

沒有組織標示。屬於個人付費或免費方案,與公司無關。

不可處理任何公司案件、業主圖檔、計算書或會議紀錄。

Caution · 細則第六章
涉及公司業務的 AI 使用,一律使用公司帳號。若你已在個人帳號裡處理過案件資料,即便事後刪除,正本仍應立即補存至公司檔案系統,並通知主管。
PART 02

介面與對話

登入後畫面長什麼樣、每個區塊在做什麼;以及「對話」這個最基本的工作單位,有哪些你必須知道的特性。

§ 03

介面總覽

登入後畫面分成三塊。左側是導覽側邊欄,中間是對話主區,右側在需要時會跳出 Artifact 預覽(見 § 07)。

3-1 · 側邊欄主要項目
項目功能
New chat開啟新對話。每次點都會建立一條獨立對話,彼此不共享記憶。
Chats過往對話列表,按時間排序。點任一對話可繼續或回查。
Projects進入 Project 列表(見 § 06)。包含你建立的、以及他人分享給你的。
Artifacts集中查看 Claude 在過往對話中產出的文件、圖、程式。
頭像 / 設定個人設定、語言、Memory 控制、登出、組織切換。
3-2 · 對話主區的關鍵元素
  • 輸入框 — 可輸入文字,也可貼上圖、表格、程式碼。Shift + Enter 換行,Enter 送出。
  • 附檔鈕(迴紋針) — 上傳檔案。可同時拖曳多檔。
  • 模型切換 — 在輸入框附近,可切 Opus / Sonnet / Haiku。
  • 工具切換 — 可開啟「Web Search」「Extended Thinking」等選項;預設關閉。
§ 04

對話(Chat):最基本的工作單位

4-1 · 一條對話 = 一個獨立脈絡

Claude 不會跨對話記住內容(除非你開啟 Memory,見 § 08)。也就是說,你在「對話 A」上傳的圖檔、討論的結論,「對話 B」完全不會知道。

實務上的選擇方式:同一件事在同一條對話裡持續處理。如果一條對話太長(超過數十輪),Claude 會逐漸忘記最早的內容,建議整理結論後另開新對話。

4-2 · 對話的命名與整理

Claude 會自動命名對話(根據第一句訊息)。命名不會很精準,建議自己改名:側邊欄對話項目上點滑鼠右鍵 → Rename。

4-3 · 對話的保存期限
Caution · 細則第三章第三項
對話內容預設保留 30 天,重要結論應另存至公司檔案系統。不要把對話紀錄當成正式工作紀錄。真的要留的結論,自己另存到公司既有檔案系統。
PART 03

附檔與 Project

這部分牽涉到「把資料交給 Claude」的兩種方式 — 一次性附檔、以及長期放在 Project 知識庫。這也是細則裡資料管理規範最密集的區域,看完務必對齊。

§ 05

附檔(Attachments):可以丟什麼、有什麼限制

5-1 · 支援的檔案格式
類別支援格式說明
文件PDF、DOCX、TXT、CSV、HTML、Markdown、RTF、ODT、EPUB會抽取文字;PDF 同時讀取圖表與版面。
試算表XLSX、CSVXLSX 支援度不及 CSV;若分析失敗,改傳 CSV。
簡報PPTX支援,但複雜版面可能遺失。
圖片JPEG、PNG、GIF、WebPBMP、TIFF、RAW 不支援。
程式碼幾乎所有常見副檔名純文字方式讀取。
5-2 · 大小與數量限制
  • 單檔上限約 30 MB。 超過會直接被拒絕。
  • 單一對話最多 20 個附檔。 超過要刪舊的才能加新的。
  • PDF 約 100 頁以內可完整解析(超過會被截斷或失敗)。
  • 圖片建議 1000×1000 像素以上;單張上限約 8000×8000。
5-3 · 上傳前一定要做的判斷
Stop · 細則第二章第四項

按下「上傳」之前,先回頭問自己這三個問題:

  1. 這份資料的正本是否仍保存於公司檔案系統?
  2. 若這份資料因任何原因外流,本公司是否仍能向業主交代?
  3. 業主若知道本案資料正被上傳至 AI 服務,是否會反對?

任一項答案為「否」或「不確定」 — 停手,問主管。

5-4 · 上傳後的去向

上傳的檔案會儲存在 Anthropic 的伺服器,作為這條對話(或 Project)的脈絡來源。Team 方案下,Anthropic 預設不會用這些內容訓練模型(詳見 § 10);但「不訓練」≠「不存在於第三方伺服器」,這兩件事要分清楚。

§ 06

Project:存放案件脈絡的工作區

6-1 · 什麼是 Project

Project 是一個「工作資料夾」的概念。每個 Project 有自己的知識庫(Knowledge Base)與專案指示(Instructions),該 Project 內所有對話都會自動參考這些內容,不必每次重貼。

適合存放的內容:該案件的規範整理、計算範本、設計準則、背景說明、業主既往來文(經判斷可上傳者)。

6-2 · 建立一個 Project
1
進入 Projects 區
側邊欄點「Projects」 → 右上角「+ New Project」。
2
命名
填寫名稱與描述。建議命名格式:案件編號 + 案件簡稱,例如 2026-XXX_新店某住宅大樓結構。
3
選擇可見性
Private(私人) 或 Shared with organization(整個組織可見)。預設選 Private,要分享給特定人再另外操作。
4
進入 Project 主頁
建立後右側即為「Project Knowledge」區,可開始上傳檔案、設定指示。
6-3 · 加入知識庫內容

在 Project 主頁右側「Project Knowledge」區點「+」,可上傳檔案或直接貼上文字。所有放進來的內容,該 Project 內每次對話都會自動參考。

Recommended · 適合放入
  • 案件用到的規範條文整理(自己整理過的)
  • 計算範本與設計準則
  • 業主背景說明、案件範圍說明
  • 過去類似案件的關鍵結論(去識別化後)
Avoid · 不適合放入
  • 極大的原始 PDF(浪費容量、難管理)
  • 與本案無關的個人資料
  • 細則第二章第三項所列的禁止項
  • 未整理過的會議錄音逐字稿原檔
6-4 · 個人 Project vs 共用 Project
Personal · 個人 Project
只有你看得到

只有你本人可以看到、可以用。其他同仁(含管理員)無法存取。

適用:個人主辦的案件、個人查核工具、規範整理筆記。

注意:帳號一旦回收(離職、停用),內容無法再被任何人取回。案件正本必須留在公司檔案系統。

Shared · 共用 Project
多人協作

透過「Share」邀請特定同仁加入。可給 Can use(可看可聊不可改)或 Can edit(可改知識庫)。

適用:多人協作大案、跨組諮詢工作。

注意:擁有權不能轉移。擁有者離職前要依細則第四章第四項完成交接。

6-5 · 分享 Project 給同仁
1
進入要分享的 Project
點右上角「Share project」(在 Project 名稱旁邊)。
2
輸入同仁 Email
必須是同組織內成員的公司網域信箱。可一次貼多筆 Email。
3
選擇權限
為每位成員選 Can use 或 Can edit。對方會收到通知,並出現在他的 Projects 列表。
Note · 補充
分享 Project 只會共享「知識庫」與「指示」,不會共享你既有的對話。對方加入後,他開的新對話自己看,你開的對話依舊只有你自己看。
PART 04

Artifact 與 Memory

Artifact 是 Claude 把「比較有份量的產出」獨立出來的方式;Memory 則是跨對話記憶的開關。前者建議學會用,後者建議先全部關閉。

§ 07

Artifacts:Claude 產出的文件、圖、程式

7-1 · 什麼是 Artifact

當 Claude 產出「比較有份量、可以獨立拿出來用」的內容時,會自動把該內容放進右側的 Artifact 視窗,而不是塞在對話文字流中。常見場景:

  • 請 Claude 撰寫一份報告草稿、會議紀錄、回覆函
  • 請 Claude 整理表格、流程圖、組織圖
  • 請 Claude 寫 Excel 巨集、Python 計算腳本、HTML 文件

Artifact 與對話分離,可以在右側獨立預覽、修改、下載。

7-2 · 操作 Artifact
  • 修改 — 在對話中直接告訴 Claude 要改什麼,它會更新 Artifact。每次修改會留下版本紀錄,右上角可切換版本。
  • 下載 — Artifact 視窗右下角有下載按鈕,可存成對應格式(HTML、TXT、PDF 等)。
  • 複製 — 也可直接複製整段內容貼到 Word、Excel 等公司既有工具中。
7-3 · Artifact 的限制與風險
  • Artifact 是 Claude 自己寫出來的靜態內容,無法即時連網取資料。
  • Artifact 內的計算、圖表、條文引用,仍需依細則第五章逐項校核(規範條號回查、計算驗算、中文用詞校對)。
  • 看起來漂亮 ≠ 正確。 排版精美的 Artifact 反而最容易讓人放鬆警覺。
Caution · 常見誤區
Claude 生成的計算書、條文引用、案例引述,有「看起來合理但其實虛構」的傾向。Artifact 越美越要警覺;直接貼進對外文件之前,務必逐項回查。
§ 08

Memory(記憶)與對話搜尋

8-1 · 兩個獨立的功能

在 Settings → Capabilities 中,可找到兩個與「跨對話記憶」相關的開關:

功能作用
Search and reference chats允許 Claude 在新對話中,自動搜尋你過去的對話內容並引用。
Generate memory from chat history允許 Claude 從你過去的對話中萃取「關於你」的摘要(角色、習慣用語、偏好格式等),自動帶入未來對話。
8-2 · 建議的預設值
Recommendation · 入門期建議
建議同仁先全部關閉,熟悉工具與細則之後,再視個人工作型態決定要不要打開。

理由:

  • 跨對話記憶會讓「上一個案件的脈絡」滲入「下一個案件的對話」,實務上不見得想要這樣。
  • 若不慎將敏感資訊納入 Memory,清除程序比清除對話麻煩。
  • 在 Project 內的脈絡傳遞,已透過「知識庫」處理;Memory 是另一條獨立通道,功能上不必要重疊。
8-3 · 檢視與清除 Memory

Settings → Capabilities → Memory,可看到 Claude 對你建立的所有 Memory 條目,逐筆刪除,或整批清空。

PART 05

用量與資料安全

用量限制決定你哪一天會「撞牆」;資料政策則是 Team 方案的核心保障 — 但保障的範圍跟一般人想的不太一樣,這一節把它說清楚。

§ 09

用量限制:會在什麼時候撞牆

9-1 · Team 方案的兩層限制

Team 方案的用量分兩層計算:

  • 5 小時滾動窗口 — 從你發出第一條訊息起算的 5 小時內,有一定的用量上限。
  • 每週上限 — 7 天內的累計用量也有上限,從每週首次使用起算。

用量是按「對話的長度、複雜度、所用模型、附檔大小」綜合計算,並非按訊息則數。Anthropic 並未公開絕對數值。

9-2 · 撞牆時的表現

接近上限時,系統會顯示提醒;達到上限後,該帳號暫時無法送出新訊息(舊對話仍可讀),需等下次重置。

9-3 · 實務建議
  • 處理大型 PDF 或長文件時,使用 Sonnet 通常足夠,不必預設用 Opus。
  • 同樣的需求,整理過再丟(把不相關段落去掉、把問題寫清楚),耗用會少很多。
  • 同一條對話越長,每次回應的耗用越大;階段性結論告一段落後,另開新對話。
§ 10

Team 方案下,你的資料怎麼被處理

10-1 · 三個關鍵事實
面向實際情況
模型訓練Anthropic 商業方案(含 Team)預設不使用你的輸入或輸出來訓練模型,除非你主動點按讚 / 倒讚回報 bug 並同意。
資料儲存位置儲存於 Anthropic 雲端伺服器。對話、附檔、Project 知識庫皆然。
員工存取Anthropic 員工原則上無法存取你的對話,除非你明確同意,或內容被安全機制標記。
10-2 · 「不訓練」和「不外洩」是兩件事

細則第一章第一項已說明這點,這裡再強調一次,因為它最容易誤解:

不訓練 · What it means
Anthropic 不拿你的內容調整模型參數

該內容不會以「模型行為」的形式影響其他用戶。Team 方案的合約承諾。

不外洩 · NOT what it means
內容仍儲存於 Anthropic 伺服器

理論上仍可能因資安事件而曝光。這是所有第三方雲端 SaaS共有的風險,不是 Claude 特有的。

Stop · 關鍵原則
業主有保密約定時,「上傳到 Claude」在法律上即構成「提供予第三方」,無論訓練與否。這是禁止上傳的依據,不要混淆這兩件事。
10-3 · 管理員可以看到什麼

Team 方案的組織擁有者 / 管理員,可以在後台看到:

  • 組織內各帳號的存在、最後活動日
  • 使用量統計(總量,非內容)
  • 共用 Project 的清單(若該 Project 設為組織可見)

但管理員無法直接讀取你的私人對話內容,也無法進入你的個人 Project 知識庫。這是 Team 方案的設計界線。

KEY TAKEAWAYS · 五件最重要的事

讀完這份介紹,你只要記得五件事就夠

1. 確認你登入的是公司帳號 — 左上角看到「築遠工程」才對。

2. 對話保留 30 天 — 重要結論自己另存,不要當正式紀錄。

3. 上傳前先問三個問題 — 正本在哪、能不能交代、業主會不會反對。

4. Memory 先關 — 熟了再開,Project 知識庫已經夠用。

5. 「不訓練」≠「不外洩」 — 業主保密約定永遠優先。

Team 方案Claude 的商業方案之一,提供共用 Project、組織管理、不訓練模型的承諾。

指示(Instructions)Project 設定中的個人化說明 — 告訴 Claude 你是誰、要做什麼、偏好。
ArtifactClaude 產出的獨立內容(文件、表、程式碼等),會顯示在右側專屬視窗。

Web Search對話內可手動開啟的選項,允許 Claude 即時搜尋網路。預設關閉。
